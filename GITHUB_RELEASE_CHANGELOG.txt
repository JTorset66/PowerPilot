## PowerPilot v1.1.2605.14550

PowerPilot v1.1.2605.14550 is the current public release build focused on plan visibility, stronger Battery plan savings, clearer window layout, and release documentation cleanup.

### Highlights

- PowerPilot plans stay Balanced-derived so all three managed plans remain visible on Modern Standby systems while Windows power mode still chooses efficiency, balanced, or maximum performance.
- Plan activation now prefers the native Windows `PowerSetActiveScheme` API with `powercfg /SETACTIVE` as fallback, avoiding normal-user activation failures seen on some Windows 11 systems.
- `PowerPilot Battery` now applies hidden Windows battery-saving policy for low-power GPU preference, PCIe maximum link saving, DC device idle, disabled DC standby networking, aggressive DC disconnected standby, display/disk/sleep timeouts, DC hibernate after 30 minutes, and disabled DC wake timers.
- Installer refresh now reapplies the current full plan policy to existing PowerPilot plans without deleting their GUIDs.
- Battery Saver text and low/reserve/critical controls were tightened so Reserve is clearly warning-only and actions sit under Low/Critical.
- The Plans tab Purpose column now uses the available width so descriptions are not clipped at normal window size.

### Documentation

- README, installer README, framework docs, function map, release checklist, SignPath notes, and signing policy now describe the current plan model and release state.
- Release tooling accepts an explicit four-part app version so `v*` tags build artifacts whose names match the tag.
- Obsolete generated chat-history logs were removed from the release commit scope.

## PowerPilot v1.1.2605.14485

PowerPilot v1.1.2605.14485 is a release-candidate build focused on Battery Saver consolidation, clearer UI text, source cleanup, and documentation alignment.

### Highlights

- Battery Saver now centralizes Windows Energy Saver threshold/brightness, low/reserve/critical battery thresholds, low/critical actions, background throttling, deep idle saver, and normal-plan restore-on-exit.
- Battery Graph marker labels cover sleep/suspend (`Z`), hibernate (`H`), wake (`W`), shutdown (`S`), startup (`P`), improper shutdown (`!`), offline (`0`), online (`1`), Energy Saver (`E`), and normal-state transitions (`N`).
- Crowded graph marker letters stack above their vertical line as white letters with black shadows, while power/offline markers keep priority.
- The app UI text was tightened across Battery Saver, PowerPilot Log, Battery Graph, Battery Stats, Power Use, Battery Test, and About.
- The installer now owns Windows startup registration for the original ordinary user. Post-install app refresh repairs plans and persists the startup preference without writing the setup-time startup entry itself.
- Installer/uninstaller post-copy commands run as the original user, while setup keeps administrator approval only for installation into Program Files.

### Documentation

- README and installer README describe the Battery Saver tab, selectable 6- to 72-hour graph windows, graph marker letters, Battery Test, Power Use, screen brightness logging, and installer-side startup behavior.
- Framework documentation and function map cover native battery reads, Battery Saver policy handling, Battery Test, Power Use, display/Energy Saver events, UI scaling, and installer/user-boundary responsibilities.

## PowerPilot v1.1

PowerPilot v1.1 adds local battery history, graphing, charging/discharging estimates, and a much smoother update/install flow while keeping the tray app on the normal user side of the installer boundary.

### Highlights

- Added Battery Graph, PowerPilot Log, Battery Stats, Power Use, Battery Test, and About tabs.
- Added retained local CSV battery history with a 168-hour cap.
- Added average and now-based battery time estimates for discharging, plus learned average time-to-target estimates for charging.
- Charging estimates use the configured `Full at` or charge-limit target, current charging watts, recent percent movement, and learned charging behavior. The live `Now` field shows `Using average` while charging to avoid a duplicate ETA.
- Added screen on/dim/off events, Energy Saver events, laptop brightness when Windows exposes it, PC power events, and short app status rows.
- Added selectable graph windows from 6 to 72 hours with event/offline markers and Energy Saver coloring.
- Added signed battery watts, Windows runtime, wear, full/design capacity, cycle count, voltage, and PowerPilot's own estimated battery cost.
- Added settings backup/import and configurable visible log columns.
- Added an in-app About tab explaining privacy, data boundaries, automatic plan follow, and included files.

### UI and wording

- Renamed visible battery terms to `Plugged in`, `Charging`, `Discharging`, `Battery %`, and `Now rate`.
- PowerPilot Log timestamps now display as `YYYY-MM-DD HH:MM:SS`; full CSV export keeps ISO-style timestamps.
- Energy Saver control now lives on the Battery Saver tab and uses `Automatic threshold` or `Battery plan always`.
- Cleanup status rows are shorter, for example `Cleaned 1 old app`.
- The old bottom status field was removed; short status messages now appear as `APP` rows in the PowerPilot Log while the Overview tab keeps the latest full action text.

### Battery and logging

- PowerPilot Log includes battery samples, app rows, screen rows, Energy Saver rows, and PC power-event rows.
- Average calculations treat sleep, hibernate, shutdown, startup, wake, improper shutdown, app restart, and installer/update close as breaks where appropriate.
- Startup drain can be learned from retained log history.
- Charging estimate learning is saved with a 32-update cap so future time-to-full estimates can start closer before fresh charging data accumulates.
- Windows charging time is shown as not reported when Windows/firmware does not provide it.
- Vendor battery calibration states that are plugged in but discharging are treated as active battery use while still showing external power is present.
- Battery Test provides guided manual discharge, Lenovo calibration reset mode, automatic vendor calibration detection, charge recovery tracking, target-time CPU load for watched drain tests, drain-helper test-mode telemetry, `BATTERY TEST` log rows, saved `.txt` reports, and copyable reports.
- Built-in laptop brightness is sampled conservatively through monitor WMI first, then `Dxva2.dll` only when the display layout is unambiguous.

### Installer and update behavior

- Build stamping now uses `1.1.YYMM.minute-of-month` and versioned `PowerPilot_V1.1...` artifact names.
- Installs newer versions side-by-side and lets the new app close older `PowerPilot_V*.exe` versions in the background.
- Same-version reinstalls close only the exact same stamped executable before overwrite.
- Update-close app rows are separated from real PC shutdown/sleep events.
- Installer cleanup removes stale old versioned app files and performs a post-install process check.
- Normal update path repairs missing PowerPilot-owned plans without recreating existing plans.

### Documentation

- README was expanded into a tab-by-tab user guide.
- Installer README was refreshed for setup/release packaging.
- Added `FUNCTION_MAP.md` to make source layout and stale-code checks easier.

## PowerPilot v1.0

Initial public release of PowerPilot, a PureBasic x64 tray utility for managing local Windows power plans.

### Highlights

- Creates and manages three PowerPilot-owned plans: Maximum, Balanced, and Battery.
- Edits the three fixed PowerPilot plans directly from the Plans tab.
- Follows Windows power mode and maps it to the matching PowerPilot plan.
- Rebuilds PowerPilot-owned plans from the selected non-PowerPilot Windows plan.
- Runs from the notification area with startup-in-tray support.
- Shows local CPU identification through CPUID.
- Shows local GPU/display-adapter information through Windows display enumeration.
- Includes a Windows installer with desktop/tray icon assets, desktop shortcut creation, startup setup, repair/uninstall support, and bundled README/license/notices files.

### Notes

- First public release of the project.
- Platform: Windows x64.
- Source: PureBasic project.
- Release binaries may be unsigned until trusted code-signing onboarding is complete.
